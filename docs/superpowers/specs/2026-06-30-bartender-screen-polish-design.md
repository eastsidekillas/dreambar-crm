# Полировка экрана бармена (KDS) — дизайн

Дата: 2026-06-30 · Ветка: `dev`

## Проблема

Экран бармена выглядит «разбросанно» рядом с ровным экраном официанта. Конкретно:

1. **Белый фон при скролле.** Когда позиций много, снизу появляется светлый фон.
   Причина: у официанта контент — единый скролл-контейнер
   `<main class="flex-1 min-h-0 overflow-y-auto">` с фоном темы. У бармена вкладки-списки
   рендерят контент без `flex-1/overflow`, поэтому при переполнении скроллится сам `<body>`,
   а его фон `--color-bg` светлый (≈ `#F5F3EE`).
   Затронуты: `orders.tab` (Заказы), `kitchen-monitor.tab` (Кухня), `bar-tables.tab` (Столы).
   `new-order.tab` и `reservations.tab` уже сделаны правильно (полноэкранные с внутренним скроллом).

2. **Дублирование и хардкод.** Цветовые хелперы продублированы
   (`urgencyColor` / `urgency` / `itemBg` / `statusStyle`), хекс-цвета (#0f172a, #1e293b,
   #334155, #f59e0b, #94a3b8, #64748b …) разбросаны по всем файлам вкладок.

## Решение

Тёмная KDS-тема сохраняется. Текущая grid-раскладка сохраняется. Только полировка, без masonry.

### 1. Раскладка и скролл (фикс бага)

Три вкладки-списка (`orders`, `kitchen-monitor`, `bar-tables`) получают единый скролл-хост:
`flex:1; min-height:0; overflow-y:auto; display:flex; flex-direction:column;` с тёмным фоном палитры.
Тогда тёмный фон тянется до низа, а скролл идёт внутри экрана, а не у `<body>`.

В оболочке `bartender.page.ts` шапка меняется `sticky top-0` → `flex-shrink-0`
(она и так вне скроллера; sticky был обходом отсутствующего скролл-контейнера).

Полноэкранные вкладки (`new-order`, `reservations`) — без изменений в раскладке.

### 2. Общий модуль `tabs/bar-ui.ts`

Централизуем (читаемость + конец дублированию):
- **Палитра** — именованные константы тёмной KDS-палитры вместо хардкод-хексов.
- `urgencyColor(min, kind)` — объединяет две версии: `'drink'` (пороги 5/10 мин) и
  `'food'` (пороги 8/15 мин) — раньше это были `urgencyColor` (orders) и `urgency` (kitchen).
- `itemBg(status)` — фон позиции по статусу кухни.
- `kitchenItemStatusChip(status)` — стиль чипа «готовится/ожидает».
- `reservationStatusStyle(status)` — стиль чипа статуса брони.
- `LIST_TAB_HOST` — строка стиля общего скролл-хоста для вкладок-списков.
- `tableChips(label)` — уже есть, остаётся.

Все вкладки импортируют отсюда; локальные копии хелперов удаляются.

### 3. Косметика «ровно»

Через токены из п.2, без смены поведения:
- Единые отступы и типографика секций.
- Одинаковые шапки тикетов (номер заказа + плашки столов + официант + таймер) в `orders` и `kitchen`.
- Единые размеры кнопок (min-height) и единые стили чипов (модификаторы / комментарии / статусы).

## Вне рамок

- `reservations-mobile.page.ts` (отдельная страница, не вкладка) — не трогаем.
- Смена темы на светлую, masonry-раскладка — отклонены пользователем.

## Проверка

- `ng build` (или существующая сборка) без ошибок.
- Ручная проверка: вкладки Заказы/Кухня/Столы с большим числом позиций — тёмный фон до низа,
  скролл внутри экрана; шапка остаётся на месте; на узком экране ничего не ломается.
